## 2. HOW TO DEPLOY (Step-by-step)

### Prerequisites
- Access to GitHub repo `lucassfreiree/autopilot`
- Can create branches, PRs, and merge (via GitHub UI or API)

### Deploy Controller (Getronics)

```
Step 1: Create branch from main
  → Branch name: claude/<descriptive-name> (or any-agent/<name>)

Step 2: Check current version
  → Read contracts/claude-session-memory.json → currentState.controllerVersion
  → New version = current + 1 patch (e.g., 3.7.0 → 3.7.1)
  → RULE: After X.Y.9, next is X.(Y+1).0 — NEVER X.Y.10

Step 3: Create/update patches in patches/ directory
  → For code changes: create patch files
  → For version bump only: use search-replace in trigger

Step 4: Edit trigger/source-change.json
  {
    "workspace_id": "ws-default",
    "component": "controller",        // or "agent"
    "change_type": "multi-file",
    "version": "<NEW_VERSION>",
    "changes": [
      {"action": "search-replace", "target_path": "package.json",
       "search": "\"version\": \"<OLD>\"", "replace": "\"version\": \"<NEW>\""},
      {"action": "search-replace", "target_path": "package-lock.json",
       "search": "\"version\": \"<OLD>\"", "replace": "\"version\": \"<NEW>\""}
    ],
    "commit_message": "chore: version bump <NEW>",
    "skip_ci_wait": false,
    "promote": true,
    "run": <PREVIOUS_RUN + 1>       // CRITICAL: must increment!
  }

Step 5: Commit + PR + Merge
  → git commit all changes
  → Create PR targeting main
  → Squash merge the PR
  → The merge AUTOMATICALLY triggers apply-source-change.yml

Step 6: Monitor
  → Check GitHub Actions for workflow run
  → Wait for all 7 stages: Setup → Session Guard → Apply & Push → CI Gate → Promote → Save State → Audit
```

## 4. HOW TO FIX COMMON ISSUES

### Pipeline failed
1. Go to Actions → find the failed run → check which stage failed
2. Read the error in job logs
3. Common fixes:
   - "duplicate tag" → increment version
   - "ESLint error" → fix code in patches
   - "TypeScript error" → check types in patches
   - "run not incremented" → increment run field in trigger
4. Fix the issue, bump run, new PR, merge, re-monitor

### Corporate CI failed
1. Trigger `ci-diagnose.yml` with component name
2. Read logs from `autopilot-state`: `state/workspaces/ws-default/ci-logs-<component>-*.txt`
3. Fix the patch, bump version, new deploy

### Dashboard stale
1. Trigger `spark-sync-state.yml` manually
2. Or wait for scheduled sync (every 1-5 min)

### State corrupted
1. Trigger `restore-state.yml` to rollback from `autopilot-backups` branch

1. **NEVER push directly to main** — always branch → PR → squash merge
2. **NEVER forget to increment `run`** in trigger — without increment, workflow won't fire
3. **NEVER deploy without monitoring** — always check workflow completion
4. **NEVER operate ws-socnew or ws-corp-1** — third-party, requires explicit authorization
5. **NEVER mix data between companies** — complete context isolation
6. **After X.Y.9, next is X.(Y+1).0** — NEVER X.Y.10
7. **JWT scope is `scope` (singular)** — NEVER `scopes` (plural)
8. **Swagger must be ASCII only** — no accented characters
9. **NEVER use search-replace with newlines** — use replace-file instead
